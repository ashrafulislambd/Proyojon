Forms & Validation

Forms are automatically managed by the framework's internal FormEngine. This handles real-time validation, dirty states, and submission buffering.

form.on('change', (field) => {
    console.log(`${field.name} updated to ${field.value}`);
});

Frequently Asked Questions

Q: Can I use this with React or Vue?

Yes, we provide official wrappers for all major JavaScript frameworks.

Q: What browsers are supported?

We support the last 2 versions of Chrome, Firefox, Safari, and Edge.

Q3: How do I change the primary color?

Update the --primary CSS variable in your root.

Q4: Is there a dark mode?

Yes, add the .dark-theme class to the body tag.

Q5: How do I report a bug?

Use our internal ticketing system or GitHub Issues.

Q6: What is the licensing?

Enterprise License for commercial use.

Q7: Where is the changelog?

Located in the CHANGELOG.md in the root folder.

Q8: Is mobile support built-in?

Yes, the framework is mobile-first.

Q9: Can I extend the core?

Yes, via the plugin architecture.

Q10: Is there a CLI?

Yes, run 'npx ent-cli' to see options.

Q11: How to handle SSR?

We support Next.js and Nuxt.js out of the box.

Q12: What is the bundle size?

Less than 15kb gzipped for the core.

Q13: Are there icons?

We recommend using Enterprise Icons (separate package).

Q14: How to clear cache?

Call Ent.cache.clear() in the console.

Q15: Is it secure?

Yes, we have built-in XSS and CSRF protection.

State Management Pattern

For complex applications, use the built-in Observable pattern to sync data across components.

        const GlobalStore = Ent.createStore({
        user: { name: 'Guest', role: 'Viewer' },
        notifications: [],
        settings: { theme: 'light' }
        });

        GlobalStore.subscribe(state => {
        console.log('State updated:', state);
        document.getElementById('username').innerText = state.user.name;
        });

Advanced Logic Hooks

The following hooks allow you to intercept the framework's lifecycle at critical points.

  • onBeforeMount: Executes before the DOM is painted.
  • onMounted: Fires once the component is ready.
  • onUpdate: Triggers on every state change.
  • onDestroy: Used for cleaning up event listeners.
Security Warning: Never use onBeforeMount to inject unsanitized HTML from a URL parameter to prevent XSS attacks.
